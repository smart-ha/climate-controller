# Climate Controller для Home Assistant

[![hacs_badge](https://img.shields.io/badge/HACS-Custom-orange.svg)](https://github.com/hacs/integration)
[![Validate](https://github.com/smart-ha/climate-controller/actions/workflows/validate.yml/badge.svg)](https://github.com/smart-ha/climate-controller/actions/workflows/validate.yml)

Кастомная интеграция, которая объединяет несколько нагревательных и охлаждающих
устройств под одной `climate`-сущностью с PID-регулированием, пресетами и
опциональным расписанием температуры по времени суток.

Одна виртуальная термостат-сущность управляет любым набором `climate.*`,
`switch` и `input_boolean` устройств, держит комнату у целевой температуры и
плавно, без резких перепадов, подводит кондиционер/обогреватель к нужному
режиму.

## Возможности

- **Одна `climate`-сущность на комнату.** HVAC-режимы `OFF` / `AUTO`, работа как
  обычный термостат в UI и во внешних интеграциях (Яндекс, Google, HomeKit —
  через `TURN_ON` / `TURN_OFF`).
- **Плавная регулировка без переохлаждения.** Целевая температура, которую
  интеграция выставляет на устройстве, привязана к **текущим показаниям датчика**
  и никогда не отклоняется от них больше чем на 4 °C — ни в охлаждении, ни в
  нагреве. Вместо «датчик 27 °C, кондиционер морозит на 16 °C» цель плавно
  сползает вместе с комнатой.
- **Пресеты.** `work`, `chill`, `sleep` с фиксированными температурами по
  умолчанию, плюс `none` — «без пресета». Ручная установка температуры
  автоматически сбрасывает активный пресет в `none`.
- **Расписание по времени суток.** Для каждого пресета — опциональные точки
  `время → температура` с линейной интерполяцией (циклической через полночь).
- **Разные типы устройств.** `climate.*` (через PID-сдвиг `set_temperature`),
  `switch` / `input_boolean` (через ШИМ — time-proportional on/off). Устройство,
  заданное и на охлаждение, и на нагрев, работает в двунаправленном режиме.
- **Флаг `passive`.** Устройство можно только наблюдать, не управляя им
  (см. ниже).
- **Восстановление состояния** после перезапуска Home Assistant
  (HVAC-режим / пресет / целевая температура).

## Установка

### Через HACS (рекомендуется)

1. HACS → ⋮ (меню в правом верхнем углу) → **Custom repositories**.
2. Добавьте `https://github.com/smart-ha/climate-controller`, категория —
   **Integration**.
3. Найдите **Climate Controller** в списке и нажмите **Download**.
4. Перезапустите Home Assistant.
5. **Настройки → Устройства и службы → Добавить интеграцию → Climate Controller**.

### Вручную

1. Скопируйте папку `custom_components/climate_controller` в директорию
   `custom_components` вашего Home Assistant.
2. Перезапустите Home Assistant.
3. Добавьте интеграцию через **Настройки → Устройства и службы → Интеграции**.

## Настройка

После добавления интеграции откройте её **Настройки (Configure)** — там
меню-ориентированный OptionsFlow:

- **Devices & sensors** — датчик температуры комнаты, порог переключения
  cooling/heating, фиксированные температуры пресетов, списки нагревательных и
  охлаждающих устройств и по паре чекбоксов **Enable** / **Passive** на каждое.
- **Sleep / Work / Chill schedule** — точки расписания для каждого пресета
  (добавление по одной, массовое редактирование, удаление).
- **Save & exit** — применяет накопленные изменения и перезагружает интеграцию.

## Флаг `passive` — устройства, которые не выключаются

Для каждого устройства в `cooling_devices` и `heating_devices` на экране
**Devices & sensors** доступен чекбокс **Passive** (по умолчанию выключен). Он
определяет, что происходит с устройством, когда контроллер уходит в `OFF`.

- **`passive = false` (обычный режим).** Устройство подчиняется зонной машине
  состояний. В `AUTO` контроллер ведёт его PID'ом, а у цели (idle-зона) —
  паркует. При переходе контроллера в `OFF` устройство **физически выключается**
  — `switch` / `input_boolean` через `turn_off`, `climate.*` через
  `set_hvac_mode=off`.
- **`passive = true` (всегда работает).** Устройство **никогда не выключается
  контроллером** и регулируется PID'ом к текущей целевой температуре на **каждом
  тике** — в `AUTO` (и в активной, и в idle-зоне), и когда контроллер в `OFF`.
  Перевод контроллера в `OFF` глушит обычные устройства, но passive-устройство
  продолжает вести комнату к цели. Основной кейс — **отопление, которое должно
  работать всегда**, поддерживая тепло даже когда комнатой не пользуются.

Регулировка идёт к той же `target_temperature`, что и у контроллера (с учётом
пресета / расписания / ручной установки). Снятие `passive` сразу возвращает
устройство под обычную зонную машину на следующем тике регулятора.
